# Raise local runtime inactivity defaults for production-scale model turns

## Summary

Raised new local runtime inactivity defaults from one minute to the existing five-minute maximum.

## Files changed

server/validation.ts; src/features/models/runtimeModel.ts; tests/runtimeModel.test.ts; tests/validation.test.ts

## Verification

Runtime focused tests, builds, lint, three real local-model trials, and complete release verification passed.

## What worked

The 35B Ollama model completed legitimate multi-minute silent generations rather than being falsely failed while GPU work was active.

## What did not work

The previous one-minute default was unsuitable for large local tool payloads even with a single executor.

## Unfinished work

Existing saved runtimes keep their explicit settings; operators may still choose shorter timeouts deliberately.
