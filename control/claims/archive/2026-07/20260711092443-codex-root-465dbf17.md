# Add bounded searchable run-history API and client paging

## Summary

Completed dependency audit for bounded run history and identified the older-run detail contract requirement.

## Files changed

None; read-only contract design only.

## Verification

Compact state omits runs after the first 100. A paged list endpoint alone would render older rows, but RunDetailPage also reads compact state and would show Run not found after following those rows.

## What worked

The end-to-end path was traced from server storage through list navigation and detail rendering before editing, preventing a broken pagination release.

## What did not work

RunDetailPage.tsx was not part of this claim, so the complete contract could not be implemented under its declared scope.

## Unfinished work

Reclaim server history/detail endpoints, RunsPage, RunDetailPage, tests, and styles together; verify page retrieval, deep links, linked audit/approval context, and large-data budgets.
