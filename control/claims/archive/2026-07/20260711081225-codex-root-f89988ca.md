# Build safe lazy workspace browser and preview

## Summary

Completed the safe lazy workspace browser and bounded preview slice of NH-0006.

## Files changed

server/index.ts, server/localTools.ts, src/api/types.ts, src/features/workspace/WorkspacePage.tsx, src/features/workspace/workspaceModel.ts, src/styles/pages.css, src/styles/responsive.css, tests/workspaceModel.test.ts, CHANGELOG.md, control/issues/items/NH-0006.md, control/issues/BOARD.md

## Verification

npm run release:verify passed control/version checks, lint, 57 tests, both TypeScript targets, and production build; npm audit --omit=dev found 0 vulnerabilities. Isolated API verified lazy root/child reads, bounded search/preview, and HTTP 400 escape blocking. Production Chrome verified preview/search/actions, ArrowRight expansion with focus retention, and 390 px no-overflow layout.

## What worked

Separate entries/search/preview contracts kept the UI truly lazy without weakening realpath checks. A bounded 200 KB reader avoids whole-file memory cost, while symlinks remain visible but explicitly blocked.

## What did not work

The legacy recursive tree endpoint remains for compatibility but is no longer used by the v2 explorer. The first boundary response used generic HTTP 500; it was remapped to an actionable 400 and reverified.

## Unfinished work

Memory management and grouped settings remain under NH-0006 and will use separate claims.
