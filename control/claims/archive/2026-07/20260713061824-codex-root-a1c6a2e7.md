# Prevent active Ollama generations from being killed by wall-clock timeout

## Summary

Replaced Ollama fixed wall-clock chat deadlines with official NDJSON streaming and response-inactivity timeouts, retained streamed tool calls/tool names, and bounded role output tokens.

## Files changed

server/runtimeAdapters.ts; server/types.ts; server/agentLoop.ts; src/features/models/ModelsPage.tsx; src/features/runs/RunDetailPage.tsx; tests/runtimeFailure.test.ts; tests/workflow.browser.test.ts; CHANGELOG.md

## Verification

Live qwen2.5-coder:14b generated for 4.33 seconds through a 1-second inactivity window; focused runtime/UI tests passed; npm run release:verify passed 303 tests plus all browser, build, smoke, and package gates.

## What worked

Ollama 0.31.2 streamed active tokens and completed normally; the adapter reconstructs content and tool calls while the executor has an 8192-token safety cap.

## What did not work

The first full matrix exposed an outdated mock completion marker and a separate control mutex race; both were corrected and the full matrix was rerun successfully.
