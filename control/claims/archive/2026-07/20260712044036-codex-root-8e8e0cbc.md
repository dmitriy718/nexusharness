# Add native Windows PowerShell quickstart bootstrap

## Summary

Added a native Windows PowerShell quickstart matching the idempotent and self-repairing Bash deployment workflow.

## Files changed

quickstart.ps1, README.md, CHANGELOG.md, and generated control archive/worklog evidence.

## Verification

PowerShell 7 and Windows PowerShell 5.1 parsing passed. PowerShell 7 completed build, schema-3 migration, and production smoke; Windows PowerShell 5.1 completed an idempotent repeat pass. Forced repair reinstalled 472 locked packages and rebuilt successfully. Native production startup reached healthy status on port 64297 with correct v2.0.0 production and lexical-memory diagnostics. Invalid ports were rejected. npm package output includes quickstart.ps1. Full release:verify passed 271 tests, lint, builds, 6 accessibility, 5 workflow, 6 visual, 4 performance, smoke, and 307-file package checks.

## What worked

Native prerequisite detection, winget Node installation path, shared Bash/PowerShell dependency fingerprint, deterministic install and repair, build, migration, smoke, help, validation, healthy-instance and port checks, readiness reporting, persistent data protection, and PowerShell 5.1/7 compatibility worked.

## What did not work

The first Windows PowerShell 5.1 pass exposed older native argument quoting behavior in node inline expressions; those expressions were replaced with cross-version-safe arguments and .NET hashing, after which the pass succeeded. The live test server was deliberately force-stopped after health verification, and the script correctly reported the external abnormal exit.

## Unfinished work

The winget Node installation branch was code-reviewed and parser-verified but was not executed because this machine already has Node 24. User-scoped winget is attempted first and may fall back to the official installer with an elevation prompt.
