# Inspect local and MCP tool boundaries for transaction migration

## Summary

Mapped local and MCP tool boundaries for staged transactional migration.

## Files changed

No source files changed; generated control claim archive and worklog only.

## Verification

Read-only call-path review of localTools.ts, mcpClient.ts, broker effect observation, approval consumption, and portable-provider guarantees.

## What worked

Deterministic file operations have enforceable workspace boundaries and can be redirected into a cell. MCP and shell paths are clearly identifiable at one dispatcher.

## What did not work

Portable worktrees cannot enforce process or network denial, and MCP remote effects cannot be rolled back. Treating these as fully transactional would be a false security claim.

## Unfinished work

Build the deterministic file-action adapter and run-summary persistence first; hardened execution must precede shell/validation migration, and MCP needs remote-effect compensation semantics.
