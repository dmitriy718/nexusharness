# Close automated WCAG 2.2 AA implementation and verification gaps

## Summary

Implemented and gated the automated WCAG 2.2 AA candidate across production routes, keyboard workflows, focus behavior, reflow, zoom, motion, contrast, and touch targets.

## Files changed

package.json; package-lock.json; src/app/AppShell.tsx; src/components/ui.tsx; src/components/useFocusTrap.ts; src/features/approvals/ApprovalsPage.tsx; src/features/audit/AuditPage.tsx; src/features/models/ModelsPage.tsx; src/features/runs/RunDetailPage.tsx; src/features/settings/SettingsPage.tsx; src/features/tools/ToolsPage.tsx; src/features/workspace/WorkspacePage.tsx; src/styles/base.css; src/styles/pages.css; src/styles/tokens.css; tests/accessibility.browser.test.ts; tests/accessibilityTokens.test.ts; UXUI/humanRequired.md; CHANGELOG.md; control/issues/items/NH-0008.md; control/issues/BOARD.md

## Verification

Perspective 1 static and model: release:verify passed control integrity, synchronized version, ESLint, 73 tests including semantic-token contrast at 4.5:1 or better, TypeScript, and production build; npm audit --omit=dev found 0 vulnerabilities. Perspective 2 rendered browser: five Playwright/axe suites scanned 18 routes plus run modes and populated setup/editor/dialog states with zero detectable A/AA violations; verified route focus/title, modal trap/return, arrow-key tabs and workspace disclosure navigation, 320px reflow, 200% content zoom, reduced motion, visible focus, 44px touch targets, and mobile navigation Escape/return.

## What worked

Axe identified invalid live-region ARIA, mismatched tree semantics, a 16px checkbox target, and a 43px touch navigation row; each was corrected and the permanent gate reproduced the fixes. Route and dialog focus behavior, live announcements, error associations, contrast tokens, forced-colors focus, and accessible disclosure navigation now have implementation and automated coverage.

## What did not work

Axe cannot conclusively compute many text contrast nodes over translucent gradient surfaces and reports them as incomplete rather than violations. The semantic token test covers the lightest declared surface, but automated tooling cannot validate lived screen-reader comprehension.

## Unfinished work

HR-002 is ready in UXUI/humanRequired.md and must pass with the owner using NVDA before NH-0008 can move to verify or done. No automated implementation work remains in this claim.
