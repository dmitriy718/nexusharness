# Persist transaction lifecycle summaries onto their owning runs

## Summary

Implemented durable run-bound execution-summary persistence and a run-scoped transaction-service factory.

## Files changed

server/store.ts; server/execution/runTransactions.ts; tests/runTransactionPersistence.test.ts; CHANGELOG.md; control/issues/items/NH-0013.md; generated control archives and worklog

## Verification

Four focused persistence tests and nine transaction-service tests passed, followed by strict lint and both production builds. Full release:verify passed 157 tests plus all accessibility, workflow, visual, performance, smoke, package, version, and control gates.

## What worked

Lifecycle summaries bind to exactly one run and cell, reject stale updates, remain mutation-safe, and advance run timestamps without regressing. The adapter composes the durable sink without widening TransactionService callback semantics.

## What did not work

The first server build exposed a return-type mismatch between the store helper and void persistence callback; the adapter now explicitly awaits and discards the returned run. No unresolved verification failures remain.

## Unfinished work

Deterministic file actions still need approval-safe cell routing. Hardened providers must precede arbitrary shell/validation migration, and MCP remote effects require compensation or remote-provider semantics.
