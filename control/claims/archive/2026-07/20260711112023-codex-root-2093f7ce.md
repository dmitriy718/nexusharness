# Add retry-safe transactional action preflight before cell execution

## Summary

Added retry-safe provider action admission before execution lifecycle transition or receipt consumption.

## Files changed

server/execution/contracts.ts; server/execution/transactionService.ts; server/execution/portableWorktreeProvider.ts; tests/transactionService.test.ts; tests/portableWorktreeProvider.test.ts; CHANGELOG.md; control issue/archive/worklog

## Verification

51 focused service/provider/contract tests, strict lint, and both production builds passed. Full release:verify passed 159 tests plus all accessibility, workflow, visual, performance, smoke, package, version, and control gates.

## What worked

Portable admission validates identities under the cell lock and supplies the isolated worktree to a trusted optional hook. Rejection leaves the cell isolated, avoids the executor, and preserves retry/rollback eligibility.

## What did not work

No unresolved failures. This hook intentionally does not claim that the portable provider isolates hostile processes or network access.

## Unfinished work

Wire the deterministic file approval adapter into this hook. Hardened providers remain required before arbitrary shell and validation migration; MCP remote effects still require compensation semantics.
