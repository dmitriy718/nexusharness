# Add idempotent self-healing quickstart bootstrap

## Summary

Added an executable, verbose, idempotent, self-repairing NexusHarness quickstart bootstrap for new-machine deployment.

## Files changed

quickstart.sh, README.md, CHANGELOG.md, and generated control worklog/archive evidence.

## Verification

Git Bash syntax/help/invalid-input checks passed. A full no-start deployment installed 472 locked dependencies, built, migrated sqlite-vec schema 3, and passed smoke. A second run skipped healthy dependencies; forced repair reinstalled successfully. Live startup reached health on a dynamic port and TERM cleanup stopped it. The preoccupied-port guard rejected an unrelated listener. Two isolated reruns cleared one unrelated transient Windows test timeout; the complete release:verify then passed 271 tests, lint, builds, 6 accessibility, 5 workflow, 6 visual, 4 performance, smoke, and 305-file package checks. Package output contains executable mode 100755 quickstart.sh.

## What worked

Node/npm detection, cross-platform path conversion, lockfile fingerprint reuse, forced dependency repair, production build, idempotent migrations, smoke, port collision safety, readiness reporting, signal forwarding, banner, and no-start modes all worked.

## What did not work

The first live-start attempt correctly refused default port 8787 because an unrelated process occupied it. The first full repository gate hit a pre-existing five-second Windows filesystem test timeout; that test passed twice in isolation and the complete gate passed on rerun. No quickstart defect remains.

## Unfinished work

Automatic Node installation is intentionally limited to Linux/macOS/WSL through pinned nvm; native Git Bash requires Node 20+ already installed. Persistent data is never automatically deleted or reset.
