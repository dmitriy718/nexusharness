# Build run-scoped portable transaction coordinator

## Summary

Built the run-scoped portable transaction coordinator for coherent reads, mutations, validation, proof, promotion, rollback, and teardown.

## Files changed

server/execution/portableWorktreeProvider.ts; server/execution/runExecutionCoordinator.ts; tests/runExecutionCoordinator.test.ts; CHANGELOG.md; control/issues/items/NH-0013.md; generated claim archive and worklog.

## Verification

Eight real-repository coordinator cases and 32 focused provider/service suites passed. Full release:verify passed with 238 tests plus every accessibility, workflow, visual, performance, smoke, package, lint, build, version, and control gate.

## What worked

Stable run cell identity, provider-validated workspace access, exact per-action leases/contracts, shared receipt chains, persisted summaries, all-configured validation proof, stale-proof invalidation, effect blocking, approval retry, commit, rollback, teardown, and persistence-failure cleanup worked.

## What did not work

An initial test attempted mutation after ready_to_commit; the transaction state machine correctly rejected it, and the regression was corrected to test mutation before verification. One Windows integration exceeded the default test timeout and was rerun successfully with a bounded 20-second limit.

## Unfinished work

The coordinator remains disconnected from agentLoop. Live selection must explicitly choose transaction or compatibility mode, manage resume/recovery, and keep arbitrary model shell and MCP calls off the portable host-validation path.
