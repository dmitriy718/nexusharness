# Replace false layout choices with onboarding and real workspace modes

## User impact

First launch promises three layouts that currently render identically and skips the setup required for a successful first run.

## Acceptance criteria

- Setup is resumable and validates runtime, models, workspace, and safety.
- Focus mode is complete.
- Studio and Orchestrate are advertised only if their distinct acceptance tests pass.
- Readiness is visible after onboarding.

## Dependencies and blockers

NH-0001 and design approval.

## Progress

The resumable six-step onboarding and all three run modes are implemented. Focus retains its narrative timeline and inspector. Studio reads the protected workspace API, provides a persisted adjustable tree, bounded read-only previews, draft context handoff, and linked run activity. Orchestrate derives its metrics and lane states from saved run, agent, approval, and audit records; interrupted work is displayed as blocked rather than live.

Automated verification covers saved-layout mapping and complete, active, queued, and blocked lane derivation. A production-browser pass verified all three enabled mode controls, 36 real root entries, a 68-line bounded package preview, four orchestration summaries, three agent assignments, linked approval activity, the six-phase Focus rail, mode persistence, and no horizontal overflow at 1440 px or 390 px. Full release verification remains the final gate for this claim.
