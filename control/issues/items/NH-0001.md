# Rebuild the responsive application shell and routing

## User impact

The current mobile interface clips core content, tablet views become excessively long, and screens cannot be deep-linked or restored.

## Evidence

See UXUI/plans.md findings UX-001 through UX-007 and the audited 390, 1024, and 1440 px renderings.

## Acceptance criteria

- No page-level horizontal clipping at 320 CSS px.
- Desktop, tablet, and mobile navigation patterns are intentional and keyboard accessible.
- Routes support refresh, deep links, Back, and Forward.
- Approvals remain reachable at every breakpoint.

## Dependencies and blockers

Design system and route architecture.

## Progress

The responsive routed shell is implemented and owner visual direction is approved under HR-001. Production tests now verify direct deep links, refresh, Back, Forward, post-navigation focus, mobile focus-contained navigation, Escape/focus return, and an always-reachable approvals link. Reflow checks pass without page-level overflow at 320 CSS px.

Thirty-six reviewed production baselines cover onboarding, dashboard, Focus, Studio, Orchestrate, and approvals at 1440×900, 1280×800, 1024×768, 768×1024, 390×844, and 360×800. Pixel-level regression checks ignore raster anti-aliasing only and fail on any substantive changed pixel.
