# Rebuild the responsive application shell and routing

## User impact

The current mobile interface clips core content, tablet views become excessively long, and screens cannot be deep-linked or restored.

## Evidence

See UXUI/plans.md findings UX-001 through UX-007 and the audited 390, 1024, and 1440 px renderings.

## Acceptance criteria

- No page-level horizontal clipping at 320 CSS px.
- Desktop, tablet, and mobile navigation patterns are intentional and keyboard accessible.
- Routes support refresh, deep links, Back, and Forward.
- Approvals remain reachable at every breakpoint.

## Dependencies and blockers

Design system and route architecture.

## Progress

Phase 3 responsive routed shell is coded and production-rendered at desktop and tablet widths. Final mobile emulation and owner visual review remain.
