# Create an accurate and inspectable run experience

## User impact

Operators cannot see most run outputs, phase cards show misleading overall state, and failed runs lack obvious recovery actions.

## Acceptance criteria

- Per-phase pending, active, complete, failed, skipped, and waiting states are correct.
- Plan, subtask, critic, validation, retrospective, and result data are inspectable.
- Run history supports search, metadata, detail, resume, retry, duplicate, and cancellation rules.
- Empty, active, successful, failed, and canceled states are tested.

## Dependencies and blockers

NH-0001 and typed API contracts.

## Progress

The Focus workflow now includes accurate backend-aligned phases, skipped validation, searchable history, output/activity inspection, retry/resume/cancel eligibility, duplicate prefill, session draft persistence, portable summaries, and legacy-object normalization.

## Verification evidence

- Model perspective: run-state, action, legacy output, summary, successful/canceled filtering, and empty-result tests pass.
- Browser perspective: a production run showed the correct completed/current phase counts, functional inspector tabs, duplicate prefill, draft persistence, named controls, and no 390 px overflow.
- Studio and Orchestrate are tracked separately as explicit Preview modes until the later mode phase.
