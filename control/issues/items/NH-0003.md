# Create an accurate and inspectable run experience

## User impact

Operators cannot see most run outputs, phase cards show misleading overall state, and failed runs lack obvious recovery actions.

## Acceptance criteria

- Per-phase pending, active, complete, failed, skipped, and waiting states are correct.
- Plan, subtask, critic, validation, retrospective, and result data are inspectable.
- Run history supports search, metadata, detail, resume, retry, duplicate, and cancellation rules.
- Empty, active, successful, failed, and canceled states are tested.

## Dependencies and blockers

NH-0001 and typed API contracts.

## Progress

Run history, detail, accurate phase rail, outputs, actions, and legacy-object normalization are present in the representative prototype. Deeper mode work remains.
