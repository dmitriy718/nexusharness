# Implement safe migration cleanup and purge lifecycle

## User impact

The portable CLI can install, launch, diagnose, and stop NexusHarness, but it cannot yet migrate checkout-local legacy state, clean disposable caches, or safely preview and purge Nexus-owned state before package removal.

## Evidence

- `easyDeploy.md` requires preview-first migration, clean, and purge workflows.
- `README.md` identifies automatic legacy migration and destructive purge as pending.
- The CLI has no `migrate`, `clean`, or `uninstall --purge` handlers.

## Acceptance criteria

- Legacy migration discovers only explicit bounded candidates, copies through staging, verifies content, preserves the source, and is idempotent.
- `nexus clean --dry-run` previews disposable cache targets and normal clean removes no durable state.
- `nexus uninstall --purge --dry-run` reports exact Nexus-owned paths, sizes, workspace exclusions, and process state without changing anything.
- Purge requires interactive confirmation or `--non-interactive --confirm-purge`, stops the service, and removes only contained Nexus-owned targets.
- `--keep-data` and `--keep-credentials` are accepted and truthfully reported.
- A configured workspace is preserved even when nested under the Nexus data root.
- Unit and compiled CLI integration tests cover traversal, symlink, nested workspace, dry-run, confirmation, migration, and idempotence.

## Dependencies and blockers

Secure OS credential-store integration is not implemented yet, so purge must report that there are no Nexus-managed credential entries rather than claim to remove them.

## Verification evidence

- `npm run release:verify` passed control/version checks, lint, 302 core tests, build, accessibility, workflow, visual, performance, production smoke, and the 47-file package audit.
- Twenty-four focused path/lifecycle/CLI tests cover nested workspace preservation, explicit-override unknown siblings, filesystem-root refusal, installed-app overlap, junction non-traversal, migration hashing/idempotence/conflicts/link refusal, cleanup preview, purge confirmation, malformed-store refusal, and doctor repair.
- The exact globally installed `@nexusharness/cli@2.0.0` tarball passed migration preview/apply/idempotence, cleanup preview, purge preview/apply, workspace and source preservation, post-purge version execution, and package uninstall from a Unicode launch path.
