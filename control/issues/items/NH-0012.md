# Bootstrap and enforce the repository control plane

## User impact

Without claims and durable work records, concurrent agents can overwrite one another and known issues can be lost.

## Acceptance criteria

- Root rules force discovery of canonical control rules.
- Claims are atomically acquired and overlapping claims are rejected.
- Heartbeat, stale takeover, release notes, archive, worklog, board, and verification work.
- Active operational files are separated from committed history.
- The bootstrap claim is archived with complete notes.
