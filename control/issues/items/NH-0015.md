# Build the portable deployment and CLI foundation

## User impact

NexusHarness currently depends on a source checkout and the current working directory. Users cannot launch, diagnose, or stop a packaged installation through a stable `nexus` command.

## Evidence

- `server/index.ts` resolves browser assets from `process.cwd()`.
- `server/version.ts` resolves package metadata from `process.cwd()`.
- `server/store.ts` defaults durable data to `./.nexusharness`.
- The root manifest has no executable mapping and is marked `UNLICENSED`.
- `easyDeploy.md` defines the target package, command, lifecycle, and release gates.

## Acceptance criteria

- Installation resources resolve relative to the installed module, not the launch directory.
- Default durable data uses per-user platform directories and still honors `NEXUSHARNESS_DATA_DIR`.
- A compiled `nexus` entry point supports version, doctor, status, open/start, and clean stop operations.
- The service publishes bounded per-user state and requires a per-launch secret for shutdown.
- Apache-2.0 licensing and direct production dependency notices exist.
- Tests launch the compiled CLI from a directory unrelated to the repository.
- npm/Homebrew publication remains blocked until the remaining `easyDeploy.md` gates pass.

## Dependencies and blockers

External npm organization/package ownership and the Homebrew tap must be completed by an authorized release owner.

## Verification evidence

- `npm run release:verify` passed the control/version gates, lint, 287 core tests, build, 6 accessibility tests, 5 browser workflow tests, 6 visual viewport suites, 4 performance tests, production smoke, and the production package audit.
- Focused deployment coverage passed 9 path/CLI tests, including concurrent launch convergence and installation-lease shutdown.
- The exact `@nexusharness/cli@2.0.0` tarball installed into a temporary global prefix and its generated `nexus.cmd` passed version, doctor, open/no-browser, status, authenticated stop, uninstall, an external data root, and a Unicode unrelated launch directory.
- The allowlisted tarball contains 46 files and is 0.24 MB before production dependencies.
