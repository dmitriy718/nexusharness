# Implement the Midnight Prism visual system

## User impact

The current interface is consistent but visually flat, weakly branded, and overuses identical bordered cards.

## Acceptance criteria

- Tokens define color, type, spacing, shape, elevation, density, focus, and motion.
- Brand effects remain restrained and never reduce dense-data legibility.
- Fonts and assets are local-first.
- Representative desktop, tablet, mobile, reduced-motion, and high-contrast states pass review.

## Dependencies and blockers

Owner-approved direction in UXUI/plans.md.

## Progress

Midnight Prism tokens, shell, atmosphere, hierarchy, status language, motion reduction, and local SVG mark are coded. The owner approved HR-001 on 2026-07-11.

## Verification

The representative visual direction is owner-approved. Reduced-motion, high-contrast, and final accessibility verification remain part of the Phase 9 release gate.
