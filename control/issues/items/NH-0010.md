# Modularize the UI and add interaction test coverage

## User impact

The single large UI file and broad untyped props make the renovation risky and leave user workflows unverified.

## Acceptance criteria

- Features and shared components are separated with typed contracts.
- Component, accessibility, end-to-end, and visual tests cover critical states.
- Existing server/tool tests remain passing.

## Dependencies and blockers

Dependency and architecture claims must not overlap feature claims.

## Progress

The former single UI file is split into app, API, shared components, feature routes, and layered styles. Interaction and accessibility test dependencies remain for later phases.
