# Modularize the UI and add interaction test coverage

## User impact

The single large UI file and broad untyped props make the renovation risky and leave user workflows unverified.

## Acceptance criteria

- Features and shared components are separated with typed contracts.
- Component, accessibility, end-to-end, and visual tests cover critical states.
- Existing server/tool tests remain passing.

## Dependencies and blockers

Dependency and architecture claims must not overlap feature claims.

## Progress

The former single UI file is split into typed app, API, shared-component, feature-route, and layered-style modules. Model/contract tests remain in place, rendered React Testing Library and user-event tests cover shared fields, live alerts, tab interaction, and modal focus behavior, and production Playwright suites cover accessibility and end-to-end route/state behavior.

The shared isolated production harness seeds realistic local data and now drives accessibility, workflow, performance, and visual suites; consequential save tests specialize that seed with server-valid model identifiers without changing reviewed visual states. Visual regression locks six representative workflows at all six plan viewports with 36 committed PNG baselines and pixel-level diff artifacts on failure. Three consequential browser workflows verify that approval rejection fails its originating run, an unreachable runtime is never persisted, and settings change only after explicit save. Two isolated process-level tests race overlapping and independent control-plane claims to verify mutex behavior. Existing server/tool/model tests remain green.
