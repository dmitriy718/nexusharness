# Create synchronized SemVer and release identity

## User impact

Version identity is duplicated and the running UI/API can silently come from different builds.

## Acceptance criteria

- package.json is the version source of truth.
- Lockfile, marketplace, API, UI, and artifacts are automatically synchronized or checked.
- UI warns about API/build mismatch.
- CHANGELOG.md, prerelease workflow, release validation, and v2.0.0 tag process are documented and tested.

## Dependencies and blockers

Valid Git repository and control plane.
