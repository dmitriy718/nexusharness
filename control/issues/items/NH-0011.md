# Create synchronized SemVer and release identity

## User impact

Version identity is duplicated and the running UI/API can silently come from different builds.

## Acceptance criteria

- package.json is the version source of truth.
- Lockfile, marketplace, API, UI, and artifacts are automatically synchronized or checked.
- UI warns about API/build mismatch.
- CHANGELOG.md, prerelease workflow, release validation, and v2.0.0 tag process are documented and tested.

## Dependencies and blockers

Valid Git repository and control plane are complete.

## Verification evidence

- Metadata perspective: version sync/check, lint, 22 tests, TypeScript compilation, and production build passed.
- Runtime perspective: a production API process and generated client HTML independently reported 2.0.0-alpha.1.
- Beta perspective: package, lockfile, marketplace, generated client metadata, API health, MCP client identity, navigation version surface, onboarding, and About now report 2.0.0-beta.1. The isolated production smoke test also verifies commit/mode identity, security headers, compact state, and bounded run-history response.
- Release workflow perspective: CHANGELOG contains a dated beta entry, MIGRATION_V2.md documents backup/compatibility/first-launch/rollback behavior, and the final v2.0.0 identity/tag remain explicitly gated behind HR-002 and HR-003.
