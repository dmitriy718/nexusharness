# Renovate workspace, memory, and settings workflows

## User impact

The workspace tree implies controls it does not have, memory is difficult to manage, and important settings are unlabeled and easy to lose.

## Acceptance criteria

- Workspace tree is lazy, searchable, expandable, keyboard accessible, and supports preview.
- Memory supports complete creation fields, search, filters, timestamps, and safe deletion.
- Settings are grouped, labeled, validated, and protected by dirty-state handling.

## Dependencies and blockers

Shared components and typed contracts.

## Progress

The workspace workflow is renovated. Root and child directories load independently through realpath-constrained endpoints; server search is breadth-first and bounded to 5,000 visited entries/100 results; file preview reads at most 200 KB and distinguishes binary/truncated content. Symbolic links remain visible but are never followed for preview. The tree supports keyboard expansion and sibling focus, while the detail surface provides breadcrumbs, metadata, line numbers, copy path, reveal parent, and an explicit run-draft context handoff. Memory and settings remain the next NH-0006 slices.

## Verification evidence

- Boundary/API perspective: an isolated store returned 36 root entries without eager `children`, loaded 8 `src` entries separately, found `tests/workspaceModel.test.ts` through bounded search, and previewed the 2,321-byte `package.json` with v2 content. A `..` listing attempt returned HTTP 400 with the workspace-escape explanation.
- Model perspective: 57 tests across seven files cover size formatting, slash normalization/breadcrumbs, safe parent reveal, explicit context-draft text, and bounded line rendering. Lint, both TypeScript targets, and the production build pass.
- Rendered perspective: production Chrome showed the configured root, 36 initial nodes, a 68-line `package.json` preview, version content, breadcrumbs, and copy/attach/parent actions. Search returned the deep model test path. ArrowRight expanded `src` to 44 visible nodes with `aria-expanded=true` and focus retained. At 390 px the shell became one 345 px column, body width was 375 px, and overflow was false.
