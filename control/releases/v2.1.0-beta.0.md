# NexusHarness v2.1.0-beta.0 release-candidate gate

**Status:** Local candidate — do not tag or publish until the external owner gates below are complete
**Version source:** `package.json`
**Expected tag:** `v2.1.0-beta.0`
**npm dist-tag:** `next`

## Repository gates

- [ ] The release-preparation commits are pushed and the GitHub worktree is reproducible from `main`.
- [ ] `npm run release:verify` passes from the candidate commit.
- [ ] `REQUIRE_CLEAN_RELEASE=1 npm run release:artifacts` (or the PowerShell equivalent) builds from a clean candidate commit.
- [ ] `npm run release:verify-artifacts` and `npm run release:smoke` pass for the exact tarball.
- [ ] Package, shrinkwrap, marketplace, API, UI, changelog, artifact manifest, and tag identities agree on `2.1.0-beta.0`.
- [ ] No active claims, secrets, private checklists, user state, logs, or generated release payloads are committed.

## Hosted npm gates

- [ ] GitHub Actions is enabled and the protected `npm-release` environment is configured.
- [ ] The `@nexusharness/cli` public package has been bootstrapped by an authorized owner if npm requires the package to exist before trusted-publisher setup.
- [ ] npm trusted publishing is restricted to `dmitriy718/nexusharness`, `release.yml`, `npm-release`, and the `npm publish` action.
- [ ] The tagged hosted matrix passes Linux x64, macOS Intel/Apple Silicon, Windows x64, and Node 20/22/24.
- [ ] The workflow publishes the exact tested tarball to npm under `next`, with npm provenance and matching GitHub artifacts.
- [ ] A clean registry install passes `nexus doctor --non-interactive`, startup, health, stop, migration preview, purge preview, and uninstall checks.

## Homebrew handoff

- [ ] The immutable GitHub release URL and SHA-256 exist before creating `Formula/nexusharness.rb` in `dmitriy718/homebrew-tap`.
- [ ] The tap formula installs the same tagged application payload, uses Homebrew-managed Node, and exposes only the `nexus` launcher.
- [ ] `brew audit --strict --online`, source install, functional `brew test`, upgrade, uninstall, and preserved-data checks pass on supported macOS and Linux hosts.

## Release rule

Never move `v2.0.0` or rebuild a published version. A failed candidate gets a new prerelease version. Homebrew publication starts only from the immutable, registry-verified GitHub release produced by this candidate workflow.
